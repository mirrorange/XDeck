## 1. 产品概述

### 1.1 产品定位

XDeck 是一款**轻量级、跨平台的图形化服务管理面板**。它融合了服务器管理面板（如宝塔面板、1Panel）的核心能力与桌面应用的便捷体验，面向开发者和运维人员，提供从本地开发到生产部署的一站式服务管理解决方案。

### 1.2 核心理念

| 理念           | 说明                                                                             |
| -------------- | -------------------------------------------------------------------------------- |
| **轻量优先**   | Rust 后端，极低资源占用，适合从树莓派到高性能服务器的各种环境                    |
| **跨平台统一** | 一套代码，桌面版（macOS/Windows/Linux）与服务器版（Headless Daemon）共用同一核心 |
| **开发者友好** | 不仅是服务器管理工具，更是开发者的本地服务编排助手                               |
| **AI 原生**    | MCP Server 组件，所有功能均可通过 AI 助手调用，实现自然语言运维                  |

### 1.3 目标用户

- **独立开发者**: 在本地快速部署和管理多个 Web 项目、后台服务
- **全栈工程师**: 管理开发/测试/生产环境中的服务，统一运维体验
- **小型团队运维**: 管理少量服务器节点，替代复杂的运维工具链
- **自托管爱好者**: 在 NAS/VPS 上部署和管理自托管服务

### 1.4 与竞品对比

| 特性         | 宝塔面板   | 1Panel | XDeck             |
| ------------ | ---------- | ------ | ----------------- |
| 跨平台桌面版 | ❌         | ❌     | ✅                |
| 本地开发支持 | ❌         | ❌     | ✅                |
| 技术栈       | Python/PHP | Go/Vue | Rust/React        |
| 资源占用     | 高         | 中     | 极低              |
| 进程守护     | 有限       | 有限   | ✅ 完整（类 PM2） |
| PTY 终端     | ❌         | ❌     | ✅                |
| AI 原生      | ❌         | ❌     | ✅ MCP            |
| 多节点管理   | 付费       | 付费   | ✅ 内置           |
| 开源         | 部分       | ✅     | ✅                |

### 2.2 技术栈

| 层级            | 技术选型                           | 说明                                                          |
| --------------- | ---------------------------------- | ------------------------------------------------------------- |
| **前端**        | Vite + React Router v7 + shadcn/ui | SPA，通过 WebSocket 与后端通信                                |
| **桌面壳**      | Tauri 2.x                          | 跨平台桌面应用，内嵌 Daemon 作为 Sidecar                      |
| **后端 Daemon** | Rust (Tokio + Axum)                | 异步运行时，高性能低占用                                      |
| **通信协议**    | WebSocket (JSON-RPC 2.0)           | 双向实时通信，支持推送                                        |
| **数据存储**    | SQLite (via rusqlite)              | 轻量嵌入式数据库，无需额外部署                                |
| **MCP Server**  | Rust (独立组件)                    | 协议桥接：对 AI 提供 stdio/SSE，对 Daemon 通过 WebSocket 通信 |

### 2.3 部署形态

#### 服务器版 (Headless Daemon)

```
xdeck-daemon [--port 9210] [--config /etc/xdeck/config.toml]
```

- 纯 CLI 启动，无 GUI 依赖
- 提供 WebSocket + REST API
- 可通过浏览器访问内置 Web UI
- 支持 systemd / launchd 管理
- 提供安装脚本一键部署

#### 桌面版 (Tauri App)

- 内嵌 Daemon 作为 Sidecar 进程
- 应用启动时自动拉起本地 Daemon
- 系统托盘常驻，后台运行
- 可同时连接本地和远程节点

#### MCP Server (独立进程)

```
xdeck-mcp [--daemon ws://localhost:9210] [--transport stdio]
```

- **对 AI 端**：通过 stdio 或 SSE 与 AI 客户端（Claude、Cursor 等）通信，遵循 MCP 协议
- **对 Daemon 端**：通过 WebSocket (JSON-RPC 2.0) 连接 Daemon，复用与 Web UI 相同的通信通道
- 桌面版中可作为 Sidecar 随 Daemon 一起启动
- 支持同时连接多个 Daemon 节点

## 3. 功能模块

### 3.1 仪表盘 (Dashboard)

**概述**: 系统状态总览，提供关键指标的实时监控。

| 功能         | 描述                                        | 优先级 |
| ------------ | ------------------------------------------- | ------ |
| 系统信息     | CPU、内存、磁盘、网络实时监控，带历史曲线图 | P0     |
| 服务状态概览 | 所有托管服务的运行状态一览                  | P0     |
| 快捷操作     | 常用操作入口（新建站点、添加服务等）        | P1     |
| 告警通知     | 资源超限、服务异常等告警                    | P1     |

| 备份与恢复     | 配置文件版本管理和回滚                     | P2     |

| 功能         | 描述                                                | 优先级 |

### 3.7 AI 与 MCP

**概述**: XDeck MCP Server 是一个**独立组件**，作为 AI 助手与 XDeck Daemon 之间的协议桥接器。它对 AI 客户端暴露标准 MCP 接口（stdio/SSE），内部通过 WebSocket (JSON-RPC 2.0) 调用 Daemon 的能力，与 Web UI 共享同一通信通道。

#### 3.7.1 MCP Server 架构

```
┌───────────────────┐      stdio/SSE       ┌───────────────────┐
│   AI Client       │◄────────────────────►│  XDeck MCP Server │
│ (Claude/Cursor/..)│      MCP Protocol    │  (独立进程)        │
└───────────────────┘                      └────────┬──────────┘
                                                    │
                                          WebSocket │ JSON-RPC 2.0
                                                    │
                                           ┌────────▼──────────┐
                                           │   XDeck Daemon    │
                                           └───────────────────┘
```

#### 3.7.2 MCP Server 功能

| 功能             | 描述                                                     | 优先级 |
| ---------------- | -------------------------------------------------------- | ------ |
| MCP 协议实现     | 实现 MCP Server，支持 stdio 和 SSE 传输                  | P0     |
| WebSocket 客户端 | 作为 WebSocket 客户端连接 Daemon，复用 JSON-RPC 2.0 协议 | P0     |
| Tools 注册       | 将 Daemon 的所有功能模块注册为 MCP Tools                 | P0     |
| 请求翻译         | 将 MCP Tool 调用翻译为 Daemon JSON-RPC 请求              | P0     |
| 事件订阅         | 订阅 Daemon 推送事件，转换为 MCP Notifications/Resources | P1     |
| Resources 暴露   | 将系统状态、日志等暴露为 MCP Resources                   | P1     |
| 权限控制         | 为 MCP 调用配置权限范围                                  | P1     |
| 多节点支持       | 可同时连接多个 Daemon 节点，AI 可跨节点操作              | P2     |


## 4. 通信协议设计

### 4.1 WebSocket 消息格式 (JSON-RPC 2.0)

**请求:**

```json
{
  "jsonrpc": "2.0",
  "id": "req-001",
  "method": "process.list",
  "params": {
    "node_id": "local",
    "filter": { "status": "running" }
  }
}
```

**响应:**

```json
{
  "jsonrpc": "2.0",
  "id": "req-001",
  "result": {
    "processes": [...]
  }
}
```

**推送事件 (Notification):**

```json
{
  "jsonrpc": "2.0",
  "method": "event.process.status_changed",
  "params": {
    "process_id": "abc123",
    "old_status": "running",
    "new_status": "stopped"
  }
}
```

### 4.2 认证机制

| 方式       | 场景       | 说明                            |
| ---------- | ---------- | ------------------------------- |
| Token Auth | 远程节点   | 首次连接时使用预共享 Token 认证 |
| Session    | Web UI     | 用户名密码登录后建立 Session    |
| mTLS       | 节点间通信 | 可选，双向 TLS 证书认证         |

## 5. 安全设计

| 安全措施     | 说明                                     | 优先级 |
| ------------ | ---------------------------------------- | ------ |
| 首次使用设置 | 首次启动时强制设置管理员账户和密码       | P0     |
| MCP 认证     | MCP Server 连接 Daemon 时使用 Token 认证 | P0     |
| HTTPS/WSS    | 生产环境中所有通信强制加密               | P0     |
| Token 认证   | 节点间通信使用不可逆 Token               | P0     |
| 操作审计     | 记录所有管理操作日志                     | P1     |
| 速率限制     | API 请求速率限制，防止暴力破解           | P1     |
| 权限分级     | 支持只读/操作员/管理员角色（多用户场景） | P2     |
| 沙箱隔离     | MCP 调用限制可执行范围                   | P1     |

## 7. 非功能性需求

### 7.1 性能要求

| 指标            | 目标          |
| --------------- | ------------- |
| Daemon 内存占用 | 空闲时 < 30MB |
| WebSocket 延迟  | < 50ms (本地) |
| 并发管理进程数  | ≥ 200         |
| 前端首屏加载    | < 2s          |
| Daemon 启动时间 | < 1s          |

### 7.2 兼容性

| 平台    | 支持版本                                            |
| ------- | --------------------------------------------------- |
| macOS   | 12+ (Intel & Apple Silicon)                         |
| Windows | 10+ (x64)                                           |
| Linux   | Ubuntu 20.04+ / Debian 11+ / CentOS 8+ (x64, ARM64) |
| Docker  | 20.10+                                              |
| Nginx   | 1.18+                                               |

### 7.3 可靠性

- Daemon 自身异常退出后可通过 systemd/launchd 自动重启
- 数据库使用 WAL 模式，防止写入中断导致数据损坏
- 进程守护状态持久化，Daemon 重启后自动恢复管理的进程

## 8. 用户体验设计原则

| 原则             | 说明                                           |
| ---------------- | ---------------------------------------------- |
| **暗色优先**     | 默认深色主题，符合开发者偏好。支持亮色主题切换 |
| **信息密度适中** | 参考 VS Code / Docker Desktop 的信息展示方式   |
| **操作即时反馈** | 所有操作通过 WebSocket 实时反馈状态            |
| **渐进式复杂度** | 常用功能一目了然，高级配置按需展开             |
| **键盘友好**     | 支持常用键盘快捷键                             |
| **响应式布局**   | 适配从 1280px 到 4K 分辨率                     |

## 附录 A: 术语表

| 术语             | 说明                                                |
| ---------------- | --------------------------------------------------- |
| **Daemon**       | XDeck 后端守护进程，负责所有服务管理逻辑            |
| **MCP Server**   | 将 Daemon 的功能通过 MCP 暴露给 AI                  |
| **Node/节点**    | 一个运行 XDeck Daemon 的实例（本地或远程）          |
| **Site/站点**    | 一个通过 Nginx 配置的网站（静态或反向代理）         |
| **Process/进程** | 一个被 XDeck 守护的应用进程                         |
| **PTY**          | 伪终端 (Pseudo Terminal)，允许交互式操作            |
| **MCP**          | Model Context Protocol，AI 模型与工具交互的标准协议 |
| **Sidecar**      | Tauri 中与主应用捆绑的独立可执行程序                |
